def greedy(coins, selected, remain):
    if remain <= 0:
        return 
    
    for i in coins[::-1]:
        if i <= remain:
            selected.append(i)
            greedy(coins, selected, remain-i)
            return
    return

def recTopDown(count, money, remaining):
    if remaining <= 0:
        return count[0]
    if count[remaining] <= 0:
        m = remaining
        for i in [remaining - k for k in money if k <= remaining]:
            m = min(recTopDown(count, money, i), m)
        count[remaining] = m+1
    return count[remaining]

def bottomUp(money, goal):
    count = [0]*(goal+1)

                minimum = temp

H. What is the running time of STEP F algorithm?¶

There will be some startup cost for this function, since the auxilliary array used to memorize answers starts out unfilled. When the algorithm begins, it must recursively iterate down to the lowerst values in our memorized array "count". The recursion will have a lot of overhead, which performance will depend on the machine/available memory allotted for the program. But theoretically, after the recursive function has done all the start-up work, for each value in the "count" will (in the worst case) be used as many times as there are values in the "money" array so that a minimum value can be chosen.

With the goal of finding the minimum bills $b$ for a specified amount $k$ that can be made with $n$ different bill values, the top-down function $N$ computes $N(k) = b$. The time complexity is $\Theta(nk)$.

I. What is the running time of the STEP G algorithm?¶

Each of the $n$ different bill values that can be combined to make change for an amount $k$ must be used for each value less than $k$ in the bottom-up approach. The two FOR loops very clearly express the time complexity, which similar to the above is $\Theta(nk)$. The main difference between the top-down and the bottom-up methods is that the overhead from the top-down solution makes it very slow for high input values.

J. Give an algorithm for printing the actual bills of the optimal change of $N(k)$. For example, for $k=118$, this function should print something like: $\$118 = \$1 + \$13 + \$13 + \$91$.¶

Because the bottom-up solution is prefereable, I made the modifications to it to supply the desired output. The trick is to keep track of which bill value was associated with the minimum variable when the inner IF statement is called. This leaves a trail for us to find which combination of bills yielded our solution.

We make a new array that is initialized the same as count array. Like the count array, the index will represent an amount of money. The entry at index $i$ represents the last bill in the minimum combination solution. This is stored when we find the minimum value of the current amount minus that bill. To find the combination, we start with the desired amount $a$ whose combination we want to know, take the value at that index $solutions[a]$, us it to reference the next value $a = a - solutions[a]$. We continue to update our index $a$, keeping track of the values at each of those indexes, until $a=0$. All of those values are the solution. See code below:

def explicitSolution(S):
    result = []
    a = len(S)-1
    while a > 0:
        result.append(S[a])
        a = a - S[a]
    return result
